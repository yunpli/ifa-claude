## H1 — DB CHECK status mapping (already in v2.2.0 via 8b358e4)

`report_model_outputs.status` has a CHECK constraint allowing only `(parsed, parse_failed, fallback_used, error)`. The schema-retry layer added in V2.2.0 (`feat(llm): top3 schema retry — 3 attempts with 2s/4s/8s backoff`, commit `4bded14`) introduced new `schema_retry_ok` / `schema_retry_partial` status strings that violated the constraint.

Three independent copies of `_persist_model_output` exist (drift accumulated over time):

- `ifa/families/macro/morning.py`
- `ifa/families/market/_common.py`
- `ifa/families/smartmoney/evening.py`

The fix maps:

```python
status_db = status
if status.startswith("schema_retry_ok"):
    status_db = "parsed"
elif status.startswith("schema_retry_partial"):
    status_db = "fallback_used"
elif status.startswith("error"):
    status_db = "error"
elif status_db not in ("parsed", "parse_failed", "fallback_used", "error"):
    status_db = "fallback_used"
```

Listed here so the fix is documented at release granularity, not just commit granularity.

## H2 — duplicate "先看这三件事" card

Originally `_client_brief.html` was a separate banner card meant to surface the top3 forward-looking actions before §01. After §01 (`_tone_card.html`) was redesigned to render the full pill + headline + top3 + summary + bullets layout, the client_brief banner became visually redundant — the same headline + top3 appeared twice.

`report.html` now no longer includes `_client_brief.html`. The TOC pills nav still renders. The §01 layout is unchanged. All four families (macro / market / asset / tech) inherit the fix automatically since they share `report.html`.

`_client_brief.html` is preserved on disk for now (no deletion) in case it's repurposed later; it's just no longer wired in.
