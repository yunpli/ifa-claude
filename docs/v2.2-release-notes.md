### A. Three new families

#### A.1 Research — single-stock financial-statement analyser

**Command**: `ifa research report --ts-code 600519.SH --analysis-type quarterly --tier deep`

**Capabilities**:
- 28 financial factors across 5 dimensions (profitability / growth / cash quality / balance / governance)
- SW L2 peer percentile via `sw_member_monthly` PIT JOIN
- Four lenses:
  - `quarterly_quick` — single quarter snapshot
  - `quarterly_deep` — up to 12 quarters comparison + YoY/QoQ
  - `annual_quick` — single annual report
  - `annual_deep` — 3-year trend with prior-year analysis

**Output**: `Stock-Analysis-{ts_code}-{YYYYMMDD}-{analysis_type}-{tier}.html`

**Deferred to V2.3**: HTTP API, Telegram, quota system, manual golden-set gates, dashboard.

### B. Reports UI overhaul (1 main + 3 aux)

#### B.1 §01 headline standardisation

Across all 9 family×slot combos (market morning/noon/evening + macro/asset/tech morning+evening), §01 now follows a uniform schema:

```jsonc
{
  "headline": "≤28 字一句话判断",
  "top3": ["前瞻盘中动作 1 ≤22字", "前瞻盘中动作 2", "前瞻盘中动作 3"],
  "summary": "≤80-100 字因果"
}
```

The "三件事" promise on every card front is now actually three forward-looking actions, **not a recap of §02**. Each top3 entry is "今天盘中要做的一件事": 哪个龙头/板块联动/哪条假设需要兑现。

**LLM compliance**: a new schema-validation retry layer (in `_safe_chat_json`) gives the LLM 3 attempts with 2s/4s/8s exponential backoff to populate missing required fields. Status codes (`schema_retry_ok_attempt2` etc) appear in `model_outputs.status`.

### C. Data correctness — 19 production-blocking bugs closed

GitHub issues #1—#19 (all closed). Highlights:

#### C.1 Slot cutoff (production AND historical replay)

| Slot | Live data source | Historical replay | Cut at |
|---|---|---|---|
| morning | T-1 EOD `index_daily` | same | – |
| noon | `rt_min_daily 5MIN` | `stk_mins 09:30→11:30` last bar | 11:30 |
| evening | `rt_min_daily 5MIN` | `index_daily` EOD = 15:00 | 15:00 |

If 14:00 generates the noon report, only morning bars (09:30→11:30) are used; afternoon bars are CUT.

#### C.2 Realtime breadth + SW realtime aggregation

- `_fetch_realtime_breadth_snapshot` (`market/data.py`): pulls `rt_k` for `6*.SH/0*.SZ/3*.SZ`, joins with `stk_limit` for limit-up/down counts. ~5,200 stocks in 3-8s.
- `compute_sw_realtime_snapshot` (`market/_sw_realtime.py` NEW): SW L1/L2 sector pct synthesised from member rt_k with MV-weighted aggregation (TuShare's `rt_min_daily`/`stk_mins` reject SW codes).
- **Fail-closed**: 3 wildcards must succeed with ≥1000 rows each + ≥3000 combined; partial failure → `None` (never falls back to stale EOD `daily`).

#### C.3 Trade-day discipline

`smartmoney.trade_cal` (TuShare mirror) is the single source of truth for "previous trading day". 8 sites previously used `report_date - timedelta(days=1)` which broke every Monday + post-holiday open. Now goes through `ifa.core.calendar.prev_trading_day(engine, date)`.

#### C.4 Pre-flight DB freshness check

`preflight_freshness_check(engine, family, expected_date, slot)` validates only tables the slot ACTUALLY reads:
- noon: `raw_daily`, `raw_index_daily`, `raw_sw_daily` (no `raw_moneyflow` — fund-flow-top is evening-only)
- evening: + `raw_moneyflow` for `fund_flow_top`

Trading-day-aware lag computation: 5/5 holiday (`expected = prev_trading_day = 4/30`) doesn't spuriously warn even if local DB latest is 4/30.

#### C.5 LLM hallucination patches

- §11 noon review hooks now `insert_judgment(...)` so evening §08 can load them (was always empty)
- §01 prompts no longer emit "早报假设未提供" filler when ctx.morning_hypotheses IS present
- Macro evening §02 hyps verifier prompt rewritten: never use "暂无法判断" as default; partial verdicts required when any market data is supplied

#### C.6 Engineering bugs found during UI overhaul

- M5: noon insert_judgment for review hooks
- MC4: macro §07 sector tag delimiter (was concatenated text)
- A2: chain template strip_prefix macro for "A股端A股端" duplicate label
- T2: tech "潜在蓄势待发标的池" section removed entirely
- T4: tech morning news lookback 24h → 36h + keyword expansion

→ Complete bug list: GitHub issues #1—#19, all closed.

## Migration guide v2.1.3 → v2.2

### Database

✅ **Alembic migrations are additive.** Existing tables unchanged. New schemas: `research.*`, `ta.*`, `stock.*`. Run:

```bash
uv run alembic upgrade head
```

This adds:
- `research.period_factor_decomposition`, `research.pdf_extract_cache`, `research.report_runs`, `research.report_sections`
- `ta.regime_daily`, `ta.candidate_tracking`, `ta.setup_metrics_daily`, `ta.event_signal_daily`, `ta.warnings_daily`, `ta.candidates_daily`, `ta.position_events_daily`, `ta.blacklist_daily`, `ta.fina_indicator_quarterly`, `ta.suspend_daily`, `ta.stk_limit_daily`
- `stock.tuning_promotion_log`

Rollback: `uv run alembic downgrade -1` per migration. Never tested across the whole stack — assume forward-only.

### Configuration

No breaking change. Existing `~/claude/ifaenv/secrets/.env` works as-is.

If you want to use the new pre-market warm cache (recommended):
```bash
# 09:25 BJT cron — see scripts/premarket_warm_cache.py
25 9 * * 1-5 cd /path/to/ifa-claude && uv run python scripts/premarket_warm_cache.py --skip-on-holiday
```

### Generated reports

⚠️ **Breaking visual change**: §01 layout, §06-§11 card system, and color scheme are different. Existing email templates / PDF distributions that snapshot a specific HTML layout will need a re-snapshot.

If you have automated regression tests on report HTML, update fixtures.

### Behavior changes (silent)

- Reports now show "—" instead of fabricating today's print when ETL hasn't run / TuShare hasn't published. Operators must read the `[freshness] ⚠` log line and the banner alert.
- Noon report no longer fetches `fund_flow_top` / `dragon_tiger` / `flows` (only used in evening). API call savings ~3 calls per noon.
- LLM costs may increase ~5-15% due to schema-retry layer (only when LLM omits required fields). Status codes in `model_outputs.status` track this.

### Operations checklist

| Item | Action |
|---|---|
| TuShare token | Unchanged. Verify daily ETL still runs. |
| Local PG cluster | `uv run alembic upgrade head` once after pull. |
| Cron schedule | Add `09:25 BJT` premarket warm cache (optional). |
| Existing 4-family reports | Re-run with v2.2 to see new UI. |
| Research family | Use `ifa research report` per spec; defer Telegram/API integration. |
| TA family | `ifa ta evening` daily + `ifa ta backtest` weekly. |
| Stock Edge | Single-stock invocation per analyst request; do NOT re-tune until V2.2.1. |

## Gap analysis (PM + full-stack lens)

Reading this section is mandatory before signing off on V2.2.

### What's solid
- ✅ All 9 family × slot reports generate successfully on baseline dates (4/27, 4/29, 4/30 verified)
- ✅ Data correctness: 19 bugs closed; staleness gates + fail-closed everywhere
- ✅ Trading-day discipline: 100% of date math goes through `ifa.core.calendar`
- ✅ UI consistency: 4 families share the same §01 schema + premium card components
- ✅ Mobile + print: dual-layout for critical tables; print CSS validated
- ✅ Schema retry: LLM compliance enforced at the integration layer
- ✅ Auditability: every LLM output persisted with status code + model version

### What's deferred to V2.2.1
- ⏳ **Stock Edge T3.2 + T3.3** (V2.2.1) — see [`docs/tuning-playbook.md`](tuning-playbook.md)
- ⏳ **Research HTTP API + Telegram + quota** (V2.3) — manual CLI works; productionisation later
- ⏳ **Multi-tenant report distribution** (V2.3+) — single-user `default` for now

### What's NOT validated yet (use cautiously)
- ⚠️ **Concurrent report generation** — `_RT_K_CACHE` module-level dict not thread-safe. Single-process sequential generation is the only tested mode. (GitHub issue #14, P-low)
- ⚠️ **Macro/asset/tech morning banner staleness** — Bug #4 wired to evening + market morning/noon; macro/asset/tech morning still rely on field-level None gates (GitHub issue #12, P-medium)
- ⚠️ **Cross-asset HK + 期货 realtime** — `fetch_cross_asset` uses EOD-only; no realtime path for HK indices / futures (GitHub issue #11, P-medium). Strict staleness gate prevents wrong data, but may show "—" for early-evening runs.
- ⚠️ **15:30 evening freshness false-positive** — pre-flight check expects today's data; if you run evening at 15:30 (TuShare EOD not yet published) you'll see warning lines that aren't actionable (GitHub issue #13, P-low).

### Open observability gaps (non-blocking)
- 📊 No SLO / SLA tracking for LLM call latency or schema-retry attempt rate.
- 📊 No cost dashboard for TuShare API usage (rt_k whole-A pull is ~8s and ~5,200 rows; rate-limit accounting manual).
- 📊 No automated regression test on rendered HTML structure (`--diff-html` flag not implemented).
- 📊 Print quality (PDF) not in CI; manual eyeball check only.
