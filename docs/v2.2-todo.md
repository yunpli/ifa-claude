## 风险与依赖

| 风险 | 影响 | 缓解 |
|------|------|------|
| Tushare 接口配额不足 | 高频测试受阻 | 测试环境用缓存优先；申请配额提升 |
| LLM gpt-5.4 偶发 5xx | 报告失败率上升 | fallback 到 gpt-5.5；section 降级机制 |
| 北交所数据缺口 | 北交所样本黄金集不达标 | 报告显式降级；不阻断 GA |
| pdfplumber 中文异常 | PDF 深读不可用 | V2.2.0 仅可选；失败不阻断主报告 |
| 黄金集人工标注耗时 | M6 / TA-M9 / SI-M11 拖期 | 提前启动，初期可降到 20 家/20 个交易日/20 case |
| `stk_factor_pro` 存储超预算 | DB 容量压力 | 方案 B 精选 80 字段，1.7 GB/年；超出则改为 7 日滚动窗口 |
| Regime 分类规则跨年代失效 | 老数据回测准确率低 | 规则参数可配置 `params/ta_v2.2.yaml`，按年代调阈值 |
| Setup edge 数据不足（新 setup） | 新 setup 无历史 edge | 给 default 中性参数，标注 "edge_data_pending" |
| 5min 全市场回填超时 | SI-M5b 阻塞 | 并发数从 4 提升到 8（仍在 Tushare 限速内）；分批分日回填 |
| Kronos-small 训练数据不平衡 | LightGBM 预测偏向特定 regime | 训练时按 regime 分层抽样；预测置信度根据样本量动态降级 |
| LightGBM 15 模型训练 7h 超 M1 散热限制 | 模型训练失败或 overheating | 单模型独立训练 + 间隔休息；夜间分批跑；可降级为 9 模型（3 horizon × 3 quantile） |
| DuckDB 与 Postgres 跨库 join 性能 | 报告生成 P99 偏高 | 大查询走 DuckDB native 优先；PG 仅用作元数据；必要时缓存到 timeframe_snapshot |
